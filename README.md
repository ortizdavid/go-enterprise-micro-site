## 🛠️ Sobre o Projeto

O **Go Enterprise Micro** não é apenas um "boilerplate". É um chassis de engenharia desenhado para eliminar 80% do trabalho repetitivo na criação de sistemas distribuídos, garantindo que o seu projeto nasce com padrões de auditoria, resiliência e escalabilidade.

### Key Technical Specs (Template Original):
* **Language:** Go 1.22+
* **Architecture:** CQRS (Command Query Responsibility Segregation)
* **Database:** PostgreSQL com migrações declarativas (Atlas HCL)
* **Messaging:** Transactional Outbox Pattern com RabbitMQ
* **Observability:** Zap Logger & Jaeger Tracing
* **Automation:** Makefile Industrial para Setup, Seed e Migrations

## 💡 Porquê investir no Go Enterprise Micro?

Como engenheiros, sabemos que o custo real de um software não está no "Hello World", mas na **manutenção** e na **evolução**. Este template foi desenhado para resolver os problemas que aparecem no 2º ano de produção, não apenas no 1º dia.

### 🎯 ROI Imediato para Developers e Empresas:
1. **Time-to-Market:** Salte a fase de configuração de infraestrutura (Auth, Logs, DB, Events) e foque apenas nas regras de negócio.
2. **Padrão Sénior Unificado:** Garante que toda a equipa escreve código seguindo os mesmos padrões de *Clean Architecture* e *SOLID*.
3. **Resiliência Nativa:** O uso de *Outbox Pattern* garante que as suas mensagens nunca se percam, mesmo que o Broker falhe temporariamente.
4. **Audit-Ready:** Logs estruturados e campos de auditoria automáticos em todas as entidades (CreatedBy, UpdatedBy, DeletedAt).
